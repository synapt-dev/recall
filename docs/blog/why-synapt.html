Why Synapt?

Local-first memory that beats cloud-dependent systems.

Layne Penney · March 2026

AI coding assistants forget everything between sessions. You debug a tricky issue on Monday, and by Wednesday your assistant has no memory it ever happened. Every agent memory system tries to solve this. Most of them require cloud APIs, send your code context to external servers, and still don't beat a well-configured local search.

Synapt is different. It runs entirely on your machine, requires no API keys, and scores higher on standardized benchmarks than systems that depend on GPT-4.

The benchmark landscape is misleading

Memory systems love publishing impressive-sounding numbers. The problem is that everyone uses different metrics, making direct comparisons impossible. Some report Jaccard similarity (token-level overlap between predicted and gold answers). Others report binary accuracy (an LLM judge says "correct" or "wrong"). Binary accuracy always produces higher numbers for the same underlying performance.

The LOCOMO benchmark (Long Conversational Memory) has become the standard evaluation for memory systems. It was introduced in the Mem0 paper using J-score (Jaccard similarity via LLM-as-Judge) as the primary metric. Here's how systems compare when using the same metric and methodology:

Synapt scores 76.04% overall with an 8B enrichment model — the highest of any system on LOCOMO. Even the local 3B version (73.38%) surpasses the Full-Context upper bound (72.90%), beating every cloud system while running entirely on a laptop.

What is Full-Context? The entire conversation history is passed directly to GPT-4 as context — no retrieval, no memory system, just brute force. It’s the theoretical upper bound: the model has access to every fact. Synapt beats it because focused retrieval surfaces only the relevant chunks — the answer model doesn’t have to find the needle in a haystack when the retrieval system has already pulled it out.

Other systems report higher headline numbers using different metrics. For example, Hindsight reports 89.61% using binary LLM-judge accuracy (not Jaccard), and MemU claims 92% without publishing evaluation code or specifying their metric. When comparing benchmarks, check the methodology — the metric matters more than the number.

What makes synapt different

Local-first, no API keys. Synapt runs on your laptop. Embeddings come from all-MiniLM-L6-v2 (90MB). Optional enrichment uses a 3B model via MLX on Apple Silicon or Ollama on Linux. Your code context never leaves your machine.

Hybrid search that catches what embeddings miss. Most memory systems rely on pure embedding similarity. That works for semantic matches but fails when you search for a specific config value, error message, or file path. Synapt fuses FTS5 full-text search with 384-dim embeddings via Reciprocal Rank Fusion. A result that keyword search misses can still surface if it's semantically similar — and vice versa.

Built for coding, not chat. LOCOMO tests personal conversation memory ("what's Maria's cat's name?"). Synapt is designed for coding sessions. It indexes tool results, file paths, error traces, and decision markers alongside conversation text. The query intent classifier routes debug queries differently from factual lookups, with tuned recency decay and embedding weights per intent.

Dynamic knowledge discovery. Some systems predefine fixed memory categories (e.g., 10 buckets like "preferences", "relationships"). Synapt discovers knowledge clusters from your actual data through temporal and semantic similarity, then consolidates them into durable knowledge nodes with confidence scores and source attribution.

Zero infrastructure. No Docker, no PostgreSQL, no cloud service. One SQLite file per project. Install with pip install synapt and you're running in seconds.

Architecture

Three retrieval paths run in parallel:

  1. BM25/FTS5 — Full-text search with configurable recency decay
  2. Embeddings — Cosine similarity over 384-dim vectors
  3. Knowledge — Durable facts extracted from session journals, searched via FTS5 + embeddings with confidence-weighted boosting

Results are merged via RRF (rank-based fusion, no score normalization needed), then interleaved with knowledge nodes and deduplicated. Query intent classification adjusts parameters automatically — debug queries weight recent sessions heavily, factual queries prioritize knowledge nodes.

Why this matters in practice: During a project, your assistant might discuss multiple options across sessions — approach A in session 3, approach B in session 5, then settle on B-with-modifications in session 7. Raw transcripts contain all three discussions equally. A naive search returns the most recent or most keyword-dense match, which may not be the decision. The knowledge layer extracts the final decision as a durable fact with source attribution, so when you search “what did we decide?”, the decision surfaces first — not the earlier deliberation that was superseded.

The metric problem in memory benchmarks

If you're evaluating memory systems, here's what to watch for:

  • J-score vs accuracy — Jaccard token overlap is strict. LLM binary accuracy is lenient. A system scoring 70% Jaccard might score 85%+ accuracy. They are not comparable.
  • Judge model — GPT-4 as judge is more generous than gpt-4o-mini. Same system, different judge, different score.
  • Answer model — Hindsight's best score uses Gemini-3 Pro for answer generation. Their open-source 20B model drops ~6 points. The memory system gets credit for the answer model's strength.
  • Dataset version — LOCOMO has different versions. Some systems evaluate on subsets.
  • Reproducibility — If the evaluation code isn't published, treat the numbers with skepticism.

Synapt's evaluation follows the Mem0 paper's methodology exactly: 10 LOCOMO conversations, 1540 QA pairs (categories 1-4), J-score via gpt-4o-mini judge. The evaluation code is in the repository.

Coding memory: where the gap widens

LOCOMO tests personal conversation memory — two friends discussing life events. But coding assistants need to remember tool output, error traces, debugging steps, and architectural decisions across sessions. We built CodeMemo to test this.

158 questions across 3 coding projects, 6 categories (factual, debug, architecture, temporal, convention, cross-session). Same gpt-4o-mini judge. Synapt leads Mem0 by +14.51 percentage points:
