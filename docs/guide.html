1. Installation

Install synapt from PyPI:

pip install synapt

For X/Twitter integration, install with the optional extra:

pip install synapt[x]

Verify the installation:

synapt --version
synapt recall stats

2. Configure Your Client

Claude Code

Add to your ~/.claude.json or project .mcp.json:

{
  "mcpServers": {
    "synapt": {
      "type": "stdio",
      "command": "synapt",
      "args": ["server"]
    }
  }
}

For development, use "args": ["server", "--dev"] to enable auto-reload on code changes. The server watches src/synapt/ and restarts the child process when .py files change.

Other MCP Clients

Any MCP-compatible client can connect via stdio. The server command is:

synapt server

Point your client's MCP configuration at this command using the stdio transport.

3. Initialize a Project

From your project root, run setup to create the index, install hooks, and configure .gitignore:

cd /path/to/your/project
synapt recall setup

This creates:

  • .synapt/recall/ — project index directory
  • .synapt/recall/index/recall.db — SQLite FTS5 + metadata database
  • Global hooks for SessionStart, SessionEnd, and PreCompact (Claude Code)

Build the initial index from your existing Claude Code transcripts:

synapt recall build

For subsequent sessions, the index rebuilds automatically via the SessionStart hook.

4. Available Tools

The MCP server exposes 14 recall tools. Here's what each does and when to use it.

Search

Tool Purpose When to use
recall_search Full search with transcript chunks, knowledge nodes, and cluster summaries Before editing unfamiliar files, when debugging errors, when making design decisions
recall_quick Fast, lightweight knowledge check (~500 token budget) Speculative checks when you're unsure if something was discussed before
recall_files Find sessions that touched a specific file path Before editing a file you haven't seen this session
recall_sessions List recent sessions with summaries To see what work happened recently
recall_context Drill down into a specific chunk or cluster When search results reference something you need more detail on
recall_timeline Chronological work arcs and session grouping To see how a project evolved over time, filter by branch or date

Knowledge Management

Tool Purpose
recall_journal Read, write, or list session journal entries (focus, done, decisions, next steps)
recall_remind Set and check cross-session reminders (one-shot or sticky)
recall_enrich LLM-powered journal enrichment — extracts structured knowledge from raw entries
recall_consolidate Extract durable knowledge nodes from enriched journals (dedup, cluster, summarize)
recall_contradict List or resolve knowledge contradictions (when facts conflict across sessions)

Index Management

Tool Purpose
recall_build Build or rebuild the transcript index (supports incremental mode)
recall_setup Initialize project directory, hooks, and gitignore
recall_stats Index statistics: chunks, sessions, date range, embeddings status, storage size

Key Parameters (recall_search)

Parameter Default Description
query — Search query (natural language or keywords)
max_chunks 5 Maximum transcript chunks to return
max_tokens 1500 Token budget for the response
half_life 60.0 Days for recency decay (0 = no decay)
depth "full" Detail level: "full", "summary", or "concise"
after / before none ISO 8601 date filters
max_sessions unlimited Limit to N most recent sessions (for faster progressive search)
threshold_ratio 0.2 Drop results below this fraction of top score

See the full parameter reference for all tools and options.

5. Plugins

Synapt has a plugin system based on Python entry points. Plugins can register additional MCP tools.

Built-in: X/Twitter

Install with pip install synapt[x]. Adds 8 tools: x_post, x_reply, x_thread, x_delete, x_timeline, x_mentions, x_get_tweet, x_search.

Requires environment variables: X_API_KEY, X_API_SECRET, X_ACCESS_TOKEN, X_ACCESS_TOKEN_SECRET.

External Plugins

Any Python package can register tools by adding an entry point in its pyproject.toml:

[project.entry-points."synapt.plugins"]
my_plugin = "my_package.server"

The module must export a register_tools(mcp) function that adds tools to the FastMCP server instance.

6. Configuration Files

Synapt reads configuration from two locations (project overrides global):

  • Global: ~/.synapt/config.json
  • Project: .synapt/recall/config.json (in project root)
{
  "models": {
    "embedding": "all-MiniLM-L6-v2",
    "summarization": "google/flan-t5-base",
    "enrichment": "mlx-community/Ministral-3-3B-Instruct-2512-4bit",
    "consolidation": "mlx-community/Ministral-3-3B-Instruct-2512-4bit",
    "reranker": "cross-encoder/ms-marco-MiniLM-L-6-v2"
  },
  "backend": "auto",
  "max_tokens": 1500
}

Environment Variable Overrides

Variable Effect
SYNAPT_SUMMARY_BACKEND Model backend: onnx, mlx, transformers, ollama
SYNAPT_EMBEDDING_MODEL Override embedding model
SYNAPT_ENRICHMENT_MODEL Override enrichment/consolidation model
SYNAPT_MAX_TOKENS Override default token budget
SYNAPT_RERANKER Enable cross-encoder reranking (true/1/yes)
SYNAPT_ONNX_THREADS Thread count for ONNX Runtime inference

7. Troubleshooting

"No index found"

Most tools return this when no index exists. Fix:

cd /path/to/project
synapt recall setup
synapt recall build

"Embeddings: unavailable"

The server falls back to BM25 keyword search when embedding models can't load. This still works but may produce less relevant results. To fix, ensure sentence-transformers is installed:

pip install sentence-transformers

No transcripts found

Synapt discovers Claude Code transcripts from ~/.claude/projects/. If you use a non-standard location, ensure transcripts are in a directory synapt can find. Run synapt recall build with --verbose to see which directories are scanned.

Hooks not firing

Check that hooks are installed in ~/.claude/settings.json:

synapt recall install-hook

This adds SessionStart, SessionEnd, and PreCompact hooks that auto-rebuild and archive transcripts.

Stale results

If search returns outdated information, rebuild the index:

synapt recall build              # Incremental (fast)
synapt recall build --no-cache   # Full rebuild

8. Performance Tuning

Token Budget

The max_tokens parameter controls how much context the server returns. Lower values are faster and cheaper; higher values provide more context.

  • 500 — Quick lookups, concise answers
  • 1500 — Default, good balance
  • 3000+ — Deep research, complex debugging

Recency Decay

The half_life parameter (in days) controls how quickly older results fade. Set to 0 to disable recency weighting entirely.

  • 7 — Strongly favor recent work
  • 60 — Default, gentle decay
  • 0 — No decay, pure relevance ranking

Progressive Search

Use max_sessions to limit search scope for faster results:

# Only search the 5 most recent sessions
recall_search(query="auth middleware", max_sessions=5)

Depth Modes

  • "full" — Complete transcript chunks with all context
  • "summary" — Condensed chunk summaries
  • "concise" — Minimal output, just the facts

Enrichment

For better knowledge extraction, enable LLM enrichment. On Apple Silicon, MLX runs Ministral 3B locally. You can also use Ollama or a cloud backend:

# Local MLX (Apple Silicon, default)
synapt recall enrich

# Via Ollama
SYNAPT_SUMMARY_BACKEND=ollama synapt recall enrich

Enrichment + consolidation produces knowledge nodes — durable facts extracted from raw conversations. These rank higher in search and power the recall_quick tool.
