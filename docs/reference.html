Intent Classification

Queries are automatically classified into intents that adjust search behavior. The classifier runs on every recall_search call (unless overridden by explicit parameters).

Intent emb_weight half_life knowledge_boost Example queries
factual 1.5 90 3.0 "what database do we use?", "what's the API endpoint?"
temporal 1.0 0 (disabled) 0.5 "what did we do last week?", "recent changes to auth"
procedural 1.2 90 2.5 "how do I deploy?", "what's the release process?"
debug 0.8 30 1.0 "why did tests fail?", "that auth middleware bug"
decision 1.5 30 0.8 "why did we choose Postgres?", "what was the decision on auth?"
exploratory 2.0 60 1.5 "tell me about the architecture", "what is recall?"
aggregation 2.0 0 (disabled) 2.5 "how many bugs this sprint?", "list all features we shipped"
general 1.0 60 2.0 Default fallback for unclassified queries

When half_life is set to the MCP default (60.0), intent classification overrides it with the intent-specific value. Pass half_life=0 explicitly to disable recency decay regardless of intent.

Recency Decay

Recency decay applies an exponential time-based weight to search results. The formula:

weight = 2 ^ (-age_days / half_life)

At half_life = 60 days:

  • Today's results: weight = 1.0
  • 60 days ago: weight = 0.5
  • 120 days ago: weight = 0.25
  • 180 days ago: weight = 0.125

The decay multiplies with relevance score, so a highly relevant old result can still outrank a weakly relevant new one.

half_life Behavior Best for
0 No decay, pure relevance Searching for timeless facts, architecture decisions
7 Aggressive recency bias Debugging, "what just happened" queries
30 Moderate recency Process/workflow questions
60 Gentle decay (default) General-purpose queries

Depth Modes

Mode Token cost Content When to use
"full" High Complete transcript chunks with full context, tool calls, and code Deep research, understanding exactly what happened
"summary" Medium Condensed chunk summaries with key points Getting the gist of past work without full detail
"concise" Low Minimal output — just the essential facts Quick checks, confirming a single detail

Config Files

Locations (project overrides global)

  • Global: ~/.synapt/config.json
  • Project: .synapt/recall/config.json

Full Schema

{
  "models": {
    "embedding": "all-MiniLM-L6-v2",
    "summarization": "google/flan-t5-base",
    "enrichment": "mlx-community/Ministral-3-3B-Instruct-2512-4bit",
    "consolidation": "mlx-community/Ministral-3-3B-Instruct-2512-4bit",
    "reranker": "cross-encoder/ms-marco-MiniLM-L-6-v2"
  },
  "backend": "auto",
  "max_tokens": 1500
}

The backend field selects the inference engine: "auto" (detect best available), "onnx", "mlx", "transformers", or "ollama".

Environment Variables

Variable Default Description
SYNAPT_SUMMARY_BACKEND auto Inference backend: onnx, mlx, transformers, ollama
SYNAPT_EMBEDDING_MODEL all-MiniLM-L6-v2 Sentence embedding model
SYNAPT_ENRICHMENT_MODEL Ministral 3B (MLX) LLM for enrichment/consolidation
SYNAPT_RERANKER_MODEL ms-marco-MiniLM-L-6-v2 Cross-encoder reranker model
SYNAPT_MAX_TOKENS 1500 Default token budget for search
SYNAPT_RERANKER false Enable cross-encoder reranking
SYNAPT_ONNX_THREADS auto Thread count for ONNX Runtime

CLI Commands

# Index management
synapt init                            # Initialize project
synapt recall setup                    # Same setup path under recall
synapt recall build                    # Incremental build
synapt recall build --rescrub          # Re-scrub secrets + full rebuild
synapt recall rescrub                  # Standalone re-scrub + rebuild
synapt recall stats                    # Index statistics

# Search
synapt recall search "your query"      # Full search
synapt recall search "query" -k 10     # More chunks
synapt recall search "query" -t 3000   # Higher token budget

# Knowledge
synapt recall journal --write          # Write journal entry
synapt recall journal --list           # List entries
synapt recall enrich                   # LLM enrichment
synapt recall consolidate              # Extract knowledge nodes

# Server
synapt server                          # Start MCP server
synapt server --dev                    # Dev mode (auto-reload)

# Hooks
synapt recall install-hook             # Install Claude Code hooks
synapt recall hook session-start       # Manual hook trigger
synapt recall hook session-end         # Archive + journal
